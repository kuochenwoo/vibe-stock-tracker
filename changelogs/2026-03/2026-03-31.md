## 2026-03-31 21:35:00

### <span style="color:#1b8f59;font-weight:700;">FEAT</span> Backend Domain And Use-Case Refactor

### STAR

#### Situation
The backend had grown around service files that mixed domain rules, provider-specific history assembly, route defaults, and direct repository coupling, which made the code harder to extend safely.

#### Task
Refactor the backend into clearer boundaries by centralizing shared constants, introducing typed domain models for query/default behavior, extracting market-history and moving-average orchestration into a use-case layer, and adding repository protocols for cleaner service dependencies.

#### Action
- Added [constants.py](/Users/guozhen_wu/Documents/vibe-code-test/backend/app/core/constants.py) to centralize shared backend defaults such as poll intervals, Yahoo fetch windows, social/news limits, and asset session defaults.
- Added [market_rules.py](/Users/guozhen_wu/Documents/vibe-code-test/backend/app/domain/market_rules.py) with a Pydantic `TickerSessionProfile` and shared asset/session inference and freshness rules.
- Added [protocols.py](/Users/guozhen_wu/Documents/vibe-code-test/backend/app/repositories/protocols.py) so moving-average and history orchestration can depend on repository interfaces instead of concrete Postgres classes.
- Added [market_history.py](/Users/guozhen_wu/Documents/vibe-code-test/backend/app/use_cases/market_history.py) and [moving_averages.py](/Users/guozhen_wu/Documents/vibe-code-test/backend/app/use_cases/moving_averages.py) to move provider-specific history and indicator assembly out of [market_service.py](/Users/guozhen_wu/Documents/vibe-code-test/backend/app/services/market_service.py).
- Updated [market.py](/Users/guozhen_wu/Documents/vibe-code-test/backend/app/models/market.py) so API query defaults now live with the domain models using Pydantic/FastAPI-typed query definitions, and rewired [markets.py](/Users/guozhen_wu/Documents/vibe-code-test/backend/app/api/routes/markets.py), [social.py](/Users/guozhen_wu/Documents/vibe-code-test/backend/app/api/routes/social.py), and [dependencies.py](/Users/guozhen_wu/Documents/vibe-code-test/backend/app/api/dependencies.py) to use the new boundaries.

#### Result
The backend now has a cleaner domain/use-case shape: shared defaults are centralized, history logic is no longer embedded in `MarketService`, route defaults are closer to the domain models, and orchestration code depends on repository protocols instead of concrete storage classes.

#### Reason
These changes were made to reduce coupling, make future provider/feed work easier to add, and keep backend behavior easier to reason about as the project grows.
