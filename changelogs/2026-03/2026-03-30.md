# 2026-03-30

## 2026-03-30 08:56:14

### Change
Refactored the app into a more maintainable project structure, introduced provider-based market data access, and replaced brittle hard-coded ticker behavior with managed tracked instruments.

### STAR

#### Situation
The backend market fetch path was failing, the project structure was too concentrated, and tracked instruments were still tied to hard-coded assumptions.

#### Task
Make the app behave like a real project by modularizing the backend, separating providers, and supporting managed tracked tickers.

#### Action
- Refactored the backend into clearer `api`, `core`, `models`, `providers`, `repositories`, and `services` layers.
- Added a provider factory and moved market access behind provider abstractions.
- Replaced hard-coded ticker placeholders with managed tracked tickers and added ticker management endpoints and frontend support.
- Updated Postman and README to match the backend contract.

#### Result
The backend became easier to extend, tracked instruments became configurable, and the app could evolve toward additional providers cleanly.

### Reason
These changes were made to move the app from a prototype structure to a maintainable project architecture.
